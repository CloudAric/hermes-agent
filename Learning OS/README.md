# Learning OS

A simple operating system for turning scattered learning inputs into reusable knowledge, projects, and client-ready assets.

## Core Principle

Learning is not collecting.
Learning is a pipeline:

Input → Distill → Practice → Ship → Archive

If a resource does not move through the pipeline, it becomes noise.

## Folder System

### 00_Inbox
Temporary holding area for anything not yet processed.

Use for:
- Links
- PDFs
- YouTube videos
- Course notes
- Screenshots
- Random ideas

Rule:
Clear this folder regularly. Nothing should live here permanently.

---

### 01_Courses
Structured learning materials.

Use for:
- Online courses
- Tutorials
- Workshops
- Learning paths
- Syllabuses

Suggested format:

01_Courses/
  Course_Name/
    README.md
    notes.md
    exercises/
    resources/

Rule:
Each course should produce either notes, a project, or a reusable template.

---

### 02_Notes
Processed knowledge in your own words.

Use for:
- Concept notes
- Frameworks
- Summaries
- Technical explanations
- Personal insights

Suggested note format:

Title:
Source:
Date:
Tags:

TLDR:

Key Ideas:
1.
2.
3.

How I can use this:

Related projects / clients:

Rule:
Do not dump raw transcripts here. Notes must be distilled.

---

### 03_Projects
Practice and output zone.

Use for:
- Mini apps
- Automation workflows
- Client demos
- Experiments
- Proof-of-concepts
- Case studies

Suggested format:

03_Projects/
  Project_Name/
    README.md
    src/
    docs/
    assets/
    lessons.md

Rule:
A learning project is complete only when it has a short README explaining what was learned and what can be reused.

---

### 04_Templates
Reusable assets created from learning.

Use for:
- Prompt templates
- SOPs
- Checklists
- Client delivery templates
- Report formats
- Code snippets
- Automation blueprints

Rule:
If you solve something twice, turn it into a template.

---

### 05_Archive
Completed, obsolete, or inactive learning materials.

Use for:
- Finished courses
- Old notes
- Deprecated resources
- Paused projects

Rule:
Archive is not trash. It is cold storage.

## Weekly Review Ritual

Once per week, review:

1. What entered 00_Inbox?
2. What should become notes?
3. What should become a project?
4. What can become a template?
5. What should be archived?

Minimum action:
Move at least one item forward in the pipeline.

## Naming Convention

Use clear names:

YYYY-MM-DD_topic-name.md
project-name/
course-name/

Examples:

2026-04-26_agentic-ai-notes.md
n8n-client-onboarding-automation/
ai-consultant-sales-framework.md

Avoid:

final.md
new note.md
random.txt
untitled folder

## Operating Rules

1. Capture fast, organize later.
2. Distill before saving into Notes.
3. Build something from important learning.
4. Convert repeated workflows into Templates.
5. Archive aggressively.
6. Learning must eventually become capability, content, code, or cashflow.

## Archer's Learning OS Loop

Daily:
Capture useful inputs into 00_Inbox.

Weekly:
Distill the best items into 02_Notes or 03_Projects.

Monthly:
Turn repeated patterns into 04_Templates.

Quarterly:
Archive dead weight.

## Golden Standard

A good Learning OS should make you faster, sharper, and more useful to clients.

If it only stores information, it has failed.
If it turns learning into action, it is alive.
